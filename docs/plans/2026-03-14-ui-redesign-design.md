# High School Circuit Simulation UI Redesign Design

**Goal:** 在不推翻现有交互模型的前提下，把项目从“功能完整的教学工具”升级为“具有清晰教学气质、强引导性和统一视觉语言的课堂实验台”。

**Context:** 当前项目已经具备稳定的三栏工作台、图表工作区、AI 助手、习题板和移动端抽屉式交互。问题不在于能力不足，而在于视觉系统偏通用、模块风格分散、首屏焦点偏弱，以及部分 UI 结构存在历史漂移。

**Primary Constraint:** 继续基于现有 `index.html` + `css/style.css` + 原生 JS 控制器体系演进，避免引入新框架和大规模交互重写。

---

## 1. 改版目标

本次改版不是“换皮”，而是一次面向教学场景的产品化升级。目标包括：

1. 建立统一且可识别的视觉系统，让用户一眼看出这是“课堂实验台”，而不是通用后台工具。
2. 提升大屏首屏的聚焦能力，让空画布在未放置元件时也具有引导性和秩序感。
3. 统一主工作区、图表、AI 助手、习题板的表面语言，使它们像一个产品，而不是几个功能模块拼接。
4. 保留当前移动端交互优势，在不牺牲操作效率的前提下进一步降低认知负担。
5. 清理 UI 结构漂移，明确“属性 / 观察 / AI / 习题”四类工作流的边界和关系。

本次设计优先级是：**教学可读性 > 操作清晰度 > 风格独特性 > 动效装饰性**。

---

## 2. 设计方向提案

### 方向 A：课堂实验台 / 仪器面板（推荐）

关键词：理化教具、测量仪表、白色实验台、功能性色块、工程秩序、局部高亮。

这是最适合当前产品的方向。它保留“工具可信度”，同时能强化教学氛围。整体不走极简 SaaS，也不走炫技科幻，而是像“数字化实验台”。记忆点来自：清洁的实验台背景、测量感强的状态标签、带仪表气质的交互控件、图表和属性面板的实验记录感。

### 方向 B：轻教材 / 讲义工作台

关键词：书页留白、注释边栏、课堂批注、内容优先、温和纸感。

优点是教育气质强，适合强化“讲解”和“练习”；缺点是电路搭建和图表观察的“仪器感”会被削弱。

### 方向 C：技术实验舱 / 轻工业控制台

关键词：工业灰蓝、标签编码、结构化边界、仪器告警、控制界面。

优点是辨识度高；缺点是对高中教学来说容易偏硬、偏专业，不如方向 A 亲和。

**建议选择方向 A。** 它最能兼顾可信度、教学友好度和独特性，也最适合在现有蓝白基础上平滑升级。

---

## 3. 品牌气质与核心体验

### 产品气质

- 不是后台
- 不是聊天应用
- 不是纯沙盒游戏
- 而是“可操作、可观察、可讲解、可练习”的数字实验台

### 用户感受目标

用户打开界面的第一反应应该是：

- “这是做电学实验的地方”
- “我知道下一步该从哪里开始”
- “这个工具很稳，适合课堂和练习”
- “图表、讲解、练习都在一个连续工作流里”

### 关键记忆点

这次改版必须强化一个记忆点：

> 中央画布像一张实验台，周围不是普通侧栏，而是“元件托盘”“测量面板”“实验助手”和“讲义板”。

这会决定整个界面的命名、样式、层级和状态表达。

---

## 4. 信息架构重构

保留三栏骨架，但重新定义每一栏的身份。

### 顶部：课堂操作层

顶部不再只是按钮容器，而是“当前实验会话”的全局控制区。建议拆成三段：

- 左侧：实验会话动作（清空、导入、导出、课堂模式）
- 中间：当前实验标题 / 空状态引导 / 最近动作反馈
- 右侧：图表、习题、AI、更多

目标是减少“按钮排成一行”的工具条感，增强“实验工作台”的结构感。

### 左侧：元件托盘

把“工具箱”重命名为“元件托盘”或“实验元件”。视觉上更像分类托盘，而不是后台菜单。分类可保留，但需要增强折叠头、当前选中态、拖拽前态和推荐元件态。

### 中央：实验画布

中央画布仍是绝对核心，但在空状态时应提供：

- 1 个简短引导标题
- 2–3 个课堂常用快捷开始动作
- 1 个示例电路入口
- 当前交互模式提示

这样大屏不会只剩“空网格”。

### 右侧：实验面板

明确恢复为真正的多工作流面板，建议采用 3 个页签：

- 属性
- 观察
- 说明

其中“说明”用于接入 AI 摘要或实验提示，而不是把 AI 完全做成独立漂浮物。

---

## 5. 视觉系统方案

### 色彩

建议从单纯的蓝白灰升级为“实验台中性色 + 功能标签色”体系：

- 主中性色：偏冷白、带轻微蓝灰 tint
- 主强调色：仪器蓝
- 成功色：实验运行绿
- 风险色：保护/熔断红
- 观察色：信号橙或电流洋红

颜色应尽量改为语义 token，而不是在组件里直接写颜色值。建议新增：

- `--surface-base`
- `--surface-elevated`
- `--surface-inset`
- `--accent-instrument`
- `--accent-observe`
- `--accent-run`
- `--accent-danger`
- `--text-strong`
- `--text-muted`
- `--stroke-soft`
- `--stroke-strong`

### 字体

当前字体安全，但性格不够。建议：

- 标题：保留兼容性强的中文无衬线，但增大字重和字距策略
- 正文：继续使用 CJK 友好的 sans
- 数值 / 状态 / 仪表标签：使用更紧凑、更“读数化”的数字风格

不建议引入强烈装饰字体，但建议让“标题”和“读数”明显区分。

### 阴影与边框

减少通用卡片阴影，改为：

- 工作区边界：细描边 + 极轻阴影
- 漂浮窗口：更明确的层级阴影
- 重要活动态：用描边和背景变化，少依赖泛光

---

## 6. 布局与空间节奏

当前的问题不是组件少，而是节奏太均匀。改版需要建立“紧—松—紧”的空间韵律。

### 桌面端

- 顶栏更薄更稳，减少大块按钮感
- 左侧托盘更像工具抽屉，分类间距拉开
- 中央画布保留大面积留白，但空状态时增加轻引导
- 右侧面板分区更明确，形成“标题—内容—控制”的层次

### 大屏

在 2K / 4K 屏下，当前界面问题是中央过空、两侧过窄。建议：

- 左侧托盘增加视觉密度，但不增加噪声
- 右侧实验面板增加状态摘要区
- 顶部中段增加“实验标题 / 当前状态 / 最近动作”

### 移动端

保留当前底部控制和抽屉方案，因为这是现有 UI 的明显优势。改进重点不在交互模型，而在视觉压缩质量：

- 抽屉标题更强
- 一级动作更少
- 运行态和模式态的颜色语义更明确
- AI 打开时与底部控制的冲突再弱化

---

## 7. 页面级改版方案

### 7.1 顶部操作区

现状更像“按钮条”。改版后建议拆成三层信息：

- 一级动作：导入、导出、课堂模式、更多
- 二级实验动作：图表、习题板、AI
- 状态信息：当前模式、是否运行、最近保存/导入反馈

视觉上采用“文字按钮 + 轻标签 + 少量主按钮”的结构，避免所有按钮都同权。

### 7.2 左侧元件托盘

每类元件保持图形预览，但要增强：

- 分类标题的识别度
- 当前待放置元件的状态表达
- 常用元件的优先展示
- 空间密度和点击命中统一

建议加入“最近使用元件”或“课堂常用”小节，把高中课堂最常见器件提前。

### 7.3 中央实验画布

空态时增加轻引导层：

- “从左侧拖入元件开始搭建”
- “试试：串联电路 / 并联电路 / 含电压表测量”

运行后，这层自动淡出。这样既不打扰熟练用户，也能照顾初学者。

### 7.4 右侧实验面板

建议重建为真正的多页签：

- 属性：编辑当前选中元件参数
- 观察：测量、图表、读数
- 说明：课堂提示、规则说明、AI 摘要

如果短期不恢复三页签，至少要把“观察”重新落到可见入口，否则当前结构会让用户感知断裂。

---

## 8. 图表工作区改版方案

图表是产品差异化能力，应该从“附加窗口”升级为“实验记录层”。

### 视觉目标

- 更像仪表记录卡，而不是通用弹窗
- 标题区更像实验记录头
- 图例更像测量通道
- 当前激活图表的边界更明确

### 交互目标

- 新建图表时有模板入口：电流-时间、电压-时间、功率-时间
- 图表空态要主动教用户“下一步怎么选信号”
- 手机端 legend 作为底部记录抽屉时，要更像系统级层，而不是临时浮层

### 结构建议

- 图表头：标题、类型、关闭、更多
- 主图区：曲线 + 当前读数
- 通道区：系列设置
- 注释区：记录本次实验说明（可选）

这样图表会更像课堂实验记录器。

---

## 9. AI 助手改版方案

当前 AI 面板功能够用，但视觉语言偏“独立聊天窗口”。建议改成“实验助手”而不是“聊天 APP”。

### 重新定位

AI 不是单独的第 4 个产品，而是实验流程里的辅助层：

- 解释当前电路
- 引导观察结果
- 生成课堂提问
- 结合习题板做讲解

### 视觉重构

- 弱化聊天气泡的社交产品感
- 强化“实验笔记 / 教学解释”的版式
- 快捷问题更像引导卡片，而不是普通 chips
- 让 AI 标头、图表头、习题板头共享一套标题语言

### 布局策略

- 桌面端：依旧允许浮动，但默认贴近右下并与右侧面板建立关联
- 移动端：优先作为上层抽屉或全宽面板，不与底部操作区抢主导权

---

## 10. 习题板改版方案

习题板是教学闭环的重要一环，但当前视觉上更接近独立编辑器。建议朝“课堂讲义板”方向推进。

### 角色升级

- 从“文本编辑器”变成“实验讲义 / 板书区”

### 设计重点

- 标题区域更明确
- 编辑/预览切换更像教学模式切换
- 工具栏降噪，保留最关键的讲义工具
- 预览区更像讲义排版，不像默认 Markdown 渲染

### 与主界面的关系

讲义板不应像盖在应用上方的另一个应用，而应像与实验同步的“旁注层”。

---

## 11. 组件系统重建建议

建议建立一套轻量设计系统，优先覆盖以下组件：

- 顶部文本按钮
- 语义按钮（运行 / 停止 / 危险）
- 页签
- 面板标题栏
- 抽屉
- 元件项
- 属性卡
- 状态标签
- 图表窗口头
- AI 引导卡
- 空状态模块

每个组件需要统一：

- 尺寸级别
- 圆角级别
- 文字层级
- 悬停 / 按下 / 激活 / 禁用状态
- 手机端最小命中尺寸

目标不是做庞大 design system，而是先把高频表面语言统一。

---

## 12. 动效策略

本项目不适合大量炫技动效，推荐“少而准”的实验台动效。

建议保留并强化的动效：

- 抽屉上滑 / 下滑
- 图表窗口激活 / 拖拽态
- 空状态淡入淡出
- 运行状态切换
- 当前模式切换

建议避免的动效：

- 弹跳
- 高强度泛光
- 大面积玻璃拟态
- 与教学无关的装饰动画

所有动效都应服务于“状态切换可感知”，而不是“看起来很酷”。

---

## 13. 文案与命名调整

建议统一界面文案风格，减少“系统术语”和“开发术语”感。

推荐方向：

- 工具箱 → 元件托盘 / 元件库
- 面板 → 实验面板
- 添加图表 → 新建观察图表
- AI 助手 → 实验助手
- 观察 → 测量与观察
- 属性 → 元件属性

文案目标是：

- 更像课堂语言
- 更少重复
- 更少纯技术命名
- 一眼知道动作结果

---

## 14. 分阶段落地路线

### Phase 1：视觉系统和结构纠偏

- 建立颜色、阴影、边框、尺寸 token
- 统一顶部、左侧、右侧三大表面样式
- 修复“属性 / 观察”结构漂移
- 增加画布空状态引导

### Phase 2：核心面板升级

- 重做右侧实验面板
- 重做元件托盘分类层级
- 重做状态栏与顶部操作区关系
- 统一运行 / 停止 / 课堂模式的语义表达

### Phase 3：高级模块统一

- 图表工作区升级为实验记录层
- AI 助手改为实验解释层
- 习题板改为课堂讲义层

### Phase 4：移动端精修

- 优化抽屉压缩态
- 优化 AI 与底部操作冲突
- 优化小屏下图表 legend 和观察入口

---

## 15. 验收标准

改版完成后，应满足以下标准：

1. 首屏在空电路状态下也能清楚表达“下一步做什么”。
2. 用户能快速理解元件搭建、属性编辑、测量观察、AI 解释、习题记录之间的关系。
3. 桌面端和移动端看起来像同一个产品，而不是两套不同界面。
4. 图表、AI、习题板不再像外挂模块，而是融入主工作流。
5. 即使不看功能，只看截图，也能看出这是“课堂实验台类产品”。

---

## 16. 最终建议

这次 UI 改版最重要的不是做更多功能，而是把已经做出来的能力组织得更像一个成熟产品。

如果只做一件事，优先做：

> **建立“课堂实验台 / 仪器面板”视觉系统，并把右侧面板重新定义为真正的实验面板。**

因为这一步会同时改善：

- 首屏识别度
- 教学引导性
- 模块统一性
- 后续图表 / AI / 习题板的整合空间

如果只追求表面美化，不先解决结构和气质问题，界面会变得更精致，但不会变得更好用，也不会更有记忆点。
