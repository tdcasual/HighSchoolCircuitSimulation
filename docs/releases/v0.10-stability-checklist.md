# v0.10 Stability Checklist

目标：在发布 v0.10 前，确保交互模式冲突风险、移动端关键路径和回归门禁均已达标。

## 1. 必跑自动化门禁

- [ ] `npm run lint`
- [ ] `npm test`
- [ ] `npm run test:e2e:wire`
- [ ] `npm run test:e2e:responsive`
- [ ] `npm run mode-conflict-matrix`

## 2. 关键产物核对

- [ ] `output/e2e/wire-interaction` 生成最新截图或失败证据
- [ ] `output/e2e/responsive-touch` 生成最新截图或失败证据
- [ ] `output/e2e/mode-conflict/matrix-summary.json` 存在且 `conflicts=0`

## 3. 交互模式稳定性验收

- [ ] 手机布局（`layout-mode-phone`）下无 `wire + endpoint-edit` 双活冲突
- [ ] 紧凑布局（`layout-mode-compact`）下无模式冲突
- [ ] 课堂模式 `standard/enhanced` 与端点编辑、布线切换行为一致
- [ ] 嵌入只读场景（embed readonly）下矩阵结果 `failures=0`

## 4. 发布记录

- [ ] 记录本次执行日期、执行人、命令输出摘要
- [ ] 记录剩余已知风险与回滚触发条件
- [ ] 将矩阵报告路径写入发布说明
